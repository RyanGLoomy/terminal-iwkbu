PENJELASAN CLASS DIAGRAM
Sistem Integrasi Terminal IWKBU

Class diagram ini menggambarkan struktur statis Sistem Integrasi Terminal IWKBU yang terdiri dari 24 class. Class User merepresentasikan akun otentikasi pengguna di Supabase Auth, yang kemudian diwarisi oleh Profile sebagai data profil pengguna dan oleh PO sebagai data perusahaan otobus. Peran pengguna dikelola melalui class Role dan UserRole yang menghubungkan Profile dengan Role secara many-to-many. Class Armada menyimpan data kendaraan milik PO beserta class ArmadaDokumen untuk berkas pendukungnya. Class Terminal, PetugasTerminal, PetugasPinSession, dan SesiPetugas mengelola operasional terminal mulai dari penugasan petugas hingga sesi pencatatan transaksi. Transaksi kendaraan dicatat melalui class KendaraanMasuk dan KendaraanKeluar yang saling terhubung. Hasil rekonsiliasi IWKBU direpresentasikan oleh class Finding beserta FindingClarification untuk klarifikasi PO dan FindingAction untuk tindak lanjut. Proses rekonsiliasi itu sendiri melibatkan class IwkbuSourceRecord sebagai data sumber, IwkbuSyncRun untuk pencatatan eksekusi, IwkbuSyncStatus untuk status per armada, dan RekonsiliasiPeriode untuk siklus periode. Class Notification, ActivityLog, SystemSettings, dan JenisKendaraan berfungsi sebagai komponen pendukung sistem. Hubungan antar class menggunakan empat jenis notasi UML yaitu inheritance atau segitiga kosong untuk pewarisan Profile dan PO terhadap User, composition atau diamond hitam untuk hubungan induk-bagian yang kuat seperti PO terhadap Armada dan Finding terhadap klarifikasinya, association atau garis biasa untuk hubungan struktital longgar, serta dependency atau garis putus-putus untuk ketergantungan tanpa kepemilikan data.
